# Commands

Command documentation is organized by top-level command group:

- `auth`
- `calendars`
- `events`

Each subcommand has its own file under `docs/commands/<command>/<sub-command>.md`.

Examples:

- [`docs/commands/auth/status.md`](/Users/bryanray/Projects/Personal/calendr/docs/commands/auth/status.md)
- [`docs/commands/calendars/get.md`](/Users/bryanray/Projects/Personal/calendr/docs/commands/calendars/get.md)
- [`docs/commands/events/upcoming.md`](/Users/bryanray/Projects/Personal/calendr/docs/commands/events/upcoming.md)
